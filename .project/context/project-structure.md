# Project Structure

## Canonical Boundaries

- `README.md` and `CONTRIBUTING.md`: public product and contribution context.
- `AGENTS.md`: repo-local operating rules for agents.
- `BOOTSTRAP.md`: Delano setup brief and Pi project starting point.
- `HANDBOOK.md`: Delano operating model installed with the runtime.
- `.project/`: delivery truth, context pack, project contracts, registries, templates.
- `.agents/`: canonical Delano runtime assets, scripts, skills, hooks, schemas, rules.
- `.delano/`: optional Delano presentation/runtime metadata from installation.
- `.codex/`: Codex adapter material generated by Delano installation.
- `.repos/`: vendored reference source, read-only unless explicitly requested.

## Runtime Areas

- `apps/server`: WebSocket server, provider registry, provider drivers, adapters, orchestration, persistence.
- `apps/web`: React web UI, provider settings, chat composer, model picker, timeline rendering.
- `apps/mobile`: Expo/React Native shell that shares client runtime logic.
- `apps/desktop`: Electron shell and packaged backend.
- `packages/contracts`: schema-only shared contracts.
- `packages/shared`: shared runtime utilities with explicit subpath exports.
- `packages/client-runtime`: shared web/mobile runtime for connection and domain operations.

## Documentation Areas

- `docs/architecture/`: provider, orchestration, connection, remote, and relay architecture.
- `docs/reference/`: scripts, workspace layout, and vocabulary.
- `docs/operations/`: CI, observability, release, and relay operations.
- `docs/providers/`: provider-specific setup guides.
