# Delano Context Pack

This folder captures repo-specific delivery context for T3 Code. It should be read before changing provider runtime, web/mobile connection behavior, remote access, or Delano project contracts.

Required files:

- `project-overview.md`: mission, active scope, and current health.
- `project-brief.md`: problem, target outcome, and scope boundaries.
- `tech-context.md`: stack, runtime constraints, and integration points.
- `project-structure.md`: repository boundaries and ownership map.
- `system-patterns.md`: architecture and delivery patterns to reuse.
- `product-context.md`: users, core flows, and constraints.
- `project-style-guide.md`: naming, documentation, and review expectations.
- `progress.md`: latest delivery state and next work.
- `gui-testing.md`: UI verification policy.

The active project contract is `.project/projects/pi-agent-support/`.
